## 🗝️ Project Overview
The **Bank System (Version 1.0)** is a console-based application designed to manage client financial records. It focuses on the core pillars of data management: **C**reate, **R**ead, **U**pdate, and **D**elete (CRUD).

### 📊 Concepts & Problems Covered (25-in-1)
This project is equivalent to solving **25+ advanced programming challenges**, focusing on system architecture and data integrity.

| # | Concept | Application in Project |
|---|---|---|
| 1 | **File Handling (fstream)** | Reading and writing client records to `Clients.txt`. |
| 2 | **Data Structures (Structs)** | Organizing client info (Account, Pin, Name, Balance). |
| 3 | **Vector Management** | Efficiently loading, searching, and storing records in memory. |
| 4 | **String Splitting (Parsing)** | Converting complex text lines into usable record objects. |
| 5 | **CRUD Logic** | Implementing full Create, Read, Update, and Delete workflows. |
| 6 | **Formatting & UI** | Using `iomanip` (`setw`) to create professional table views. |
| 7 | **Input Validation** | Preventing duplicate account numbers and ensuring data integrity. |
| 8 | **Menu-Driven Interface** | Building a structured navigation system with Enums. |
| 9 | **Persistent State** | Ensuring data remains saved even after closing the program. |
| 10 | **Modular Programming** | Separating logic into specialized functions for maintainability. |

## 🚀 Key Features
* **Persistent Storage:** All data is stored in a `.txt` file, ensuring no data loss between sessions.
* **Professional UI:** Utilizes horizontal headers and formatted columns for a clean user experience.
* **Search & Update:** Instantly find any client by account number and modify their details.
* **Safe Deletion:** Implements a confirmation logic before removing records from the system.
* **Error Handling:** Guards against invalid inputs and duplicate data entry.
