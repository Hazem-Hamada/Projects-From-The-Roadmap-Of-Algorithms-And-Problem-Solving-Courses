## 🗝️ Project Overview
The **Bank System Extension** transforms a simple data-entry tool into a functional financial application. It introduces a nested menu system where users can perform complex transactions like deposits and withdrawals, which are then synchronized with the permanent data file (`Clients.txt`).

### 📊 Concepts & Problems Covered (30-in-1)
This project is equivalent to solving **30+ advanced programming challenges**. It demonstrates mastery over system architecture and data flow.

| # | Concept | Application in Project |
|---|---|---|
| 1 | **Advanced Transaction Logic** | Managing deposits and withdrawals with real-time balance updates. |
| 2 | **Nested Menu Systems** | Navigating between the Main Menu and a specialized Transactions Menu. |
| 3 | **Data Synchronization** | Ensuring file records match in-memory vector data after every transaction. |
| 4 | **Mathematical Algorithms** | Calculating total bank balances and individual account adjustments. |
| 5 | **Reference Passing** | Using `&` to modify system states and vectors across multiple functions. |
| 6 | **Record Conversion** | Sophisticated parsing of strings to objects and vice versa using delimiters. |
| 7 | **UI State Management** | Dynamic screen clearing and title headers for different system modules. |
| 8 | **Robust Search Patterns** | Finding and validating account numbers before allowing sensitive operations. |

## 🚀 Key Features
* **Full Transaction Module:** Support for Deposit and Withdrawal with built-in balance logic.
* **Total Balances Report:** A summary view that aggregates all wealth managed by the system.
* **Persistent Data Integrity:** Every transaction is instantly saved to the `Clients.txt` file.
* **Enhanced Navigation:** Easy "Go Back" functionality to switch between system layers seamlessly.
* **Professional Formatting:** Tabulated data views using `iomanip` for high readability.
