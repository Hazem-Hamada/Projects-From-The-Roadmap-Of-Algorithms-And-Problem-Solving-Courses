## 🗝️ Project Overview
The **Math Quiz Game** is designed to test and improve mathematical speed and accuracy. It features a fully customizable experience where the user controls the difficulty, operation types, and the number of challenges. 

### 📊 Concepts & Problems Covered (20-in-1)
This project isn't just a game; it's a comprehensive exercise equivalent to solving **20+ individual programming problems**. 

| # | Concept | Application in Project |
|---|---|---|
| 1 | **Random Number Generation** | Generating operands based on selected difficulty levels. |
| 2 | **Handling User Input** | Securely reading and validating integer values. |
| 3 | **Conditional Logic** | Real-time answer checking and score adjustment. |
| 4 | **Arithmetic Operations** | Implementation of Addition, Subtraction, Multiplication, and Division. |
| 5 | **Structs for Organization** | Using `stQuizz` and `stQuestionInfo` to manage game state. |
| 6 | **Working with Enums** | Defining `enQuestionsLevel` and `enOPType` for clean logic. |
| 7 | **Dynamic Mixed Levels** | Logic to randomly assign difficulty and operations for "Mix" modes. |
| 8 | **Conditional Formatting** | Changing screen colors based on pass/fail status. |
| 9 | **Screen & Sound Calls** | Utilizing `system("cls")` and `\a` (alert sound) for UX. |
| 10 | **Boolean Logic Flags** | Tracking pass/fail status and round results. |

*(Also covers: Function return values, String manipulation, Loops, and Input validation).*

## 🚀 Key Features
* **Four Difficulty Levels:** Easy (1-10), Medium (10-50), Hard (50-100), and a "Mix" mode for a real challenge.
* **Six Operation Modes:** Full support for Add, Sub, Mult, Div, and a "Mix" operation mode.
* **Division Guard:** Built-in logic to handle and prevent division by zero errors.
* **Performance Tracking:** Detailed report showing Right vs. Wrong answers and overall Pass/Fail status.
* **Developer Profile:** Includes an integrated "About Me" module showcasing the developer's background and goals.
