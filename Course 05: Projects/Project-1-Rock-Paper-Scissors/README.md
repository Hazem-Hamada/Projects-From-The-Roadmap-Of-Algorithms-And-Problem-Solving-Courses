## 🗝️ Project Overview
Instead of just a simple game, this implementation focuses on **Clean Code**, **Modular Design**, and **User Experience**. It mimics a real-world application workflow, including input validation, automated opponent logic (Simple AI), and visual feedback.

### 📊 Concepts & Problems Covered (20-in-1)
This project is equivalent to solving **20 distinct programming challenges**. Here is a breakdown of the core concepts applied:

| # | Concept | Application in Project |
|---|---|---|
| 1 | **Generating Random Numbers** | Used for the Computer's choice logic. |
| 2 | **Handling User Input** | Reading and validating player choices (1-3). |
| 3 | **Working with Enums** | Defining `enGameChoise` and `enGameWinner` for readable logic. |
| 4 | **Using Structures (Structs)** | Managing `stRoundInfo` and `stGameResults`. |
| 5 | **String Manipulation** | Converting Enum values into readable names (e.g., "Stone"). |
| 6 | **Creating Functions** | Reusable logic for round winners, game flow, and UI. |
| 7 | **System Calls** | Using `system("cls")` and `system("color")` for dynamic UI. |
| 8 | **Divide and Conquer** | Breaking the game into 15+ specialized functions. |
| 9 | **Debugging & Formatting** | Printing structured final results with proper alignment. |
| 10 | **Replay Functionality** | Implementing `do-while` loops for continuous gameplay. |

*(Note: Covers many more including Boolean logic, tracking scores, and edge case handling).*

## 🚀 Key Features
* **Customizable Length:** Play anywhere from 1 to 10 rounds.
* **Dynamic UI:** The screen color changes based on the outcome (Green for Win, Red for Loss, Yellow for Draw).
* **Intelligent Feedback:** Audio alerts (`\a`) triggered upon losing a round to the computer.
* **Scalable Architecture:** Built using **Structs** and **Enums**, making it easy to extend (e.g., adding "Lizard Spock" mode).
